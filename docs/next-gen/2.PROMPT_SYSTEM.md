# Prompt System (Full-Context) — Design Spec

## 1. Purpose
Prompt System cung cấp khả năng:
- Quản lý **Prompt Templates** có versioning, tags, params schema.
- Dựng **Context Snapshot** từ Supabase (wealth + portfolio + ledger + pricing + settings).
- Gửi snapshot lên ChatGPT để nhận **phân tích/khuyến nghị**.
- Lưu lại toàn bộ dữ liệu chạy (inputs/outputs/errors) để **audit, tái lập, so sánh**.

> Lưu ý: hệ thống **không giới hạn dữ liệu** gửi lên ChatGPT. Vì vậy bắt buộc có cơ chế đóng gói và chia nhỏ (packaging/chunking) để tránh fail do độ dài.

---

## 2. Core Principles
### 2.1 Full-context by default
- Không áp dụng redaction bắt buộc.
- Người dùng có thể bật/tắt các phần dữ liệu qua `context_recipe_json` của template.

### 2.2 Reproducibility (bắt buộc)
Mỗi lần chạy phải lưu:
- `as_of_ts`
- `template_id` + `template_version`
- `snapshot_json` (đúng cái đã gửi)
- `dataset_hash` (hash snapshot) để phát hiện khác biệt
- `run_target` (chatgpt_web | api)

Không có snapshot => run không hợp lệ.

### 2.3 Token/length resilience
Vì full-context có thể rất lớn:
- Snapshot được đóng gói theo nhiều lớp (layered context)
- Runner hỗ trợ multi-part message (chunking) với cơ chế ACK gate.

## 3. Data Model (Supabase tables)
### 3.1 prompt_templates
- `id (uuid)`
- `user_id (uuid)`
- `name (text)`
- `version (text | semver)`
- `tags (text[])`
- `template_text (text)` — prompt khung có placeholders
- `params_schema_json (jsonb)` — schema input params
- `context_recipe_json (jsonb)` — quy tắc dựng snapshot
- `is_active (bool)`
- `created_at, updated_at`

### 3.2 prompt_runs
- `id (uuid)`
- `user_id (uuid)`
- `template_id (uuid)`
- `created_at (timestamptz)`
- `as_of_ts (timestamptz)`
- `target (text)` — chatgpt_web | api
- `status (text)` — queued | running | succeeded | failed | partial
- `dataset_hash (text)` — sha256(snapshot_json)
- `error_code (text, nullable)`
- `error_detail (text, nullable)`
- `meta_json (jsonb)` — tabId, latency, selector_version, etc.

### 3.3 prompt_run_inputs
- `run_id (uuid pk/fk)`
- `snapshot_json (jsonb)` — FULL context đã đóng gói
- `metrics_json (jsonb)` — các metrics tổng hợp (optional)
- `build_log_json (jsonb)` — log dựng snapshot (counts, sizes, layers)
- `created_at`

### 3.4 prompt_run_outputs
- `id (uuid)`
- `run_id (uuid)`
- `role (text)` — assistant/system/tool
- `content (text)`
- `order (int)`
- `captured_at (timestamptz)`
- `source (text)` — chatgpt_dom | api
- `meta_json (jsonb)` — tokens_est, part_no, etc.

### 3.5 prompt_run_links (optional)
- `run_id`, `asset_id?`, `account_id?`, `tx_id?`

### 3.6 prompt_ratings (optional)
- `run_id`, `rating`, `usefulness`, `followup_action`, `notes`

## 4. Context Snapshot Design
### 4.1 Layered Context (không giới hạn nhưng có cấu trúc)
Snapshot là JSON gồm các lớp:

- `layer0_rules`
  - output schema requirements (Trade Cards, Watchlist, Triggers…)
  - checklist & guardrails (risk rules, constraints)
- `layer1_metrics`
  - net worth, allocation, exposure, cash, pnl, drawdown, risk usage…
- `layer2_portfolio`
  - holdings, positions, recent price snapshots, watchlist, trade plans
- `layer3_ledger`
  - ledger theo chế độ: last_30d / last_90d / all
- `layer4_appendices`
  - raw dumps, extra logs, errors, settings, etc.

### 4.2 Context Recipe (template-controlled)
`context_recipe_json` cho phép tùy biến phạm vi dữ liệu:
- `include_accounts: true/false`
- `include_assets: true/false`
- `include_positions: true/false`
- `include_prices_history_days: number`
- `include_fx_history_days: number`
- `include_ledger: none | last_30d | last_90d | last_1y | all`
- `include_raw_tables: true/false`
- `include_errors_logs: true/false`
- `max_rows_per_table: number | null` (null = unlimited)
- `sort_order`: by_ts_desc / by_ts_asc

> Vì “không giới hạn”, `max_rows_per_table` có thể để null. Nhưng runner sẽ phải chunk.

### 4.3 Snapshot Envelope
Ví dụ khung snapshot:

```json
{
  "as_of_ts": "2026-01-30T12:00:00+07:00",
  "base_currency": "VND",
  "layer0_rules": {...},
  "layer1_metrics": {...},
  "layer2_portfolio": {...},
  "layer3_ledger": {...},
  "layer4_appendices": {...},
  "build_meta": {
    "template_id": "...",
    "template_version": "2.3.0",
    "row_counts": {...},
    "estimated_chars": 123456
  }
}
5. Run Pipeline (End-to-end)
5.1 Steps

User selects template + provides params.

Context Builder pulls dữ liệu từ Supabase theo context_recipe_json.

Build snapshot (layered JSON).

Store prompt_runs (status=queued/running).

Store prompt_run_inputs (snapshot_json, build logs).

Execute:

Web runner: gửi snapshot lên ChatGPT qua content script (chunked)

API runner: gọi Edge Function (optional)

Capture output:

Append vào prompt_run_outputs theo thứ tự.

Mark run status succeeded/failed/partial.

(Optional) Parse output → create/update trade_plans/watchlist/research_notes.

5.2 Dataset Hashing

dataset_hash = sha256(canonical_json(snapshot_json))

Canonicalization: stable key ordering, normalized whitespace.

6. Output Contract (khuyến nghị chuẩn hóa)

Dù model trả text tự do, hệ thống nên khuyến nghị format:

Market/Wealth Regime Summary

Portfolio Actions

Trade Cards (Entry/Stop/Targets/RR/Time-stop/Invalidations/Sizing)

Watchlist + Triggers

Risks & Missing Data

Output không bắt buộc JSON, nhưng nên có heading cố định để parser dễ trích.

7. Failure Modes & Handling

CONTEXT_TOO_LARGE: snapshot vượt ngưỡng 1 message → chunking

TAB_NOT_READY: content script chưa inject được

SELECTOR_CHANGED: UI ChatGPT đổi, không bấm send/capture được

RATE_LIMIT: chờ/báo lỗi

PARTIAL_CAPTURE: gửi được nhưng capture thiếu

Mọi lỗi đều ghi vào prompt_runs.error_* và errors module.

8. Non-negotiables

Snapshot input MUST be stored for every run.

Web runner MUST support multi-part chunking.

Output MUST be stored incrementally (append) để không mất dữ liệu khi crash.
