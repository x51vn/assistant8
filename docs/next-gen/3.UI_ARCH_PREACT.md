# UI Architecture (Preact) — Side Panel Spec

## 1. Overview
UI chạy trong Chrome Extension Side Panel, built bằng:
- Preact
- @preact/signals (state management)
- preact-router (optional)
- supabase-js client
- Realtime subscriptions ở UI (không chạy trong background)

Mục tiêu:
- Wealth dashboard, ledger, portfolio views
- Prompt templates, run wizard, run history
- Debug/audit: snapshot viewer + diff viewer

## 3. State Management
### 3.1 Stores using signals
- `authStore`: session, user profile, base currency
- `wealthStore`: accounts/assets/ledger views, net worth snapshots
- `portfolioStore`: holdings/pnl/watchlist/trade plans
- `promptsStore`: templates, runs, run status, run outputs
- `uiStore`: selected tab, filters, loading flags, errors banner

### 3.2 Data fetching model
- Query Supabase for initial load
- Realtime channels to refresh views:
  - prices updates
  - prompt_run status updates
  - prompt_run_outputs appended rows
  - ledger/transactions changes

---

## 4. Key Screens & UX Contracts
### 4.1 Dashboard
- Net worth (as-of) + trend chart
- Allocation by asset class/currency/account
- Alerts: stale prices, missing valuations, FX missing

### 4.2 Ledger
- Transactions list (filters: date range, account, asset, type)
- Transaction detail: splits/lines
- Reconciliation flags: imbalanced tx, duplicate external_id

### 4.3 Portfolio
- Derived holdings list, P&L summary
- Watchlist + trade plans
- Price snapshots + notes/catalysts

### 4.4 Prompts
- Template list + version history
- Template editor (text + params schema + context recipe)
- Run wizard
- Run history

### 4.5 Prompt Run Wizard (critical)
Flow:
1) Select template
2) Set params
3) Preview context recipe
4) Build snapshot (preview JSON, row counts, estimated size)
5) Execute (web runner/api)
6) Live output view (stream appended outputs)

### 4.6 Snapshot Viewer
- JSON viewer with:
  - search by key
  - collapse/expand layers
  - copy/export snapshot

### 4.7 Diff Viewer
- Compare two snapshots:
  - highlight changed keys/values
  - show row count differences

---

## 5. Messaging & Integration
UI communicates with:
- `background` via `chrome.runtime.sendMessage` for:
  - authenticated operations requiring middleware
  - run prompt web runner triggers
  - tab management (ensure ChatGPT tab exists)
- Supabase directly for:
  - CRUD data
  - realtime subscriptions

**Rule:** UI giữ realtime subscriptions; background không giữ websocket.

---

## 6. Performance & Bundle Considerations
- Side panel needs small bundle size:
  - Lazy-load pages (dynamic import)
  - Avoid heavy chart libs nếu không cần
- Virtualized tables cho ledger lớn
- Background operations for heavy compute:
  - snapshot building can run in background OR UI, but UI phải có progress state

---

## 7. Error Handling UI
- Global error banner (from `errors` table or runtime errors)
- Module-level retry buttons
- Run wizard shows:
  - build snapshot errors
  - runner errors (tab not ready, selector changed, rate limit)
  - partial output state (recover/resume)

---

## 8. Non-negotiables (UI)
- Must display run status transitions in realtime
- Must allow viewing exact snapshot sent to ChatGPT
- Must allow viewing outputs in ordered append-only timeline
